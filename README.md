# Cognitive Biases Generate Endogenous Volatility in Welfare

This repository contains the simulation framework, analysis pipelines, and figure generation notebooks for the paper **"Cognitive biases generate endogenous volatility in welfare"** (Dupont & Roy, 2026).

## Overview

The framework compares Expected Utility Theory (EUT), Prospect Theory (PT), and Cumulative Prospect Theory (CPT) within a coupled feedback system of human and financial capital. Agents choose each period between a **save** action (retain wealth, risk health decline) and an **invest** action (pay a health cost, risk health improvement). Optimal policies are computed via value iteration. 5,000 agents are then simulated forward for 5,000 steps.

The `beta` parameter (discount factor) is required throughout. The paper reports results for two values: **0.95** and **0.999**. All commands below must be run once per desired `beta` value.

## 1. Initialization (Run First)

Generate the shared initial agent states and LHS parameter samples. These are prerequisites for all experiments.

### 1a. Initial Agent States

Generates 5,000 agent starting positions (wealth, health) via Latin Hypercube Sampling, saved to `data/initial_states.pickle`.

```bash
uv run generate_initial_states.py --n-agents 5000 --grid-size 200 --seed 42
```

### 1b. LHS Parameter Samples

Generates parameter samples for the global sensitivity analysis. Each model samples over a different active parameter set:

- **EUT**: `alpha`, `rate`, `A` (gamma and lambda are fixed at 1)
- **PT**: `alpha`, `lambda`, `rate`, `A` (gamma is fixed at 1)
- **CPT**: `alpha`, `gamma`, `lambda`, `rate`, `A`

Parameter ranges are defined in `generate_parameter_sample.py`.

```bash
uv run generate_parameter_sample.py --n-samples 1024 --model eut --seed 42
uv run generate_parameter_sample.py --n-samples 1024 --model pt  --seed 42
uv run generate_parameter_sample.py --n-samples 1024 --model cpt --seed 42
```

Samples are saved to `data/<model>/<model>_samples.pickle`.

## 2. Core Experiments (LHS)

These simulations form the backbone of the analysis. Run for each desired `beta` value. For CPT, a probability weighting function must be specified via `--cpt-weight-function` (`kt`, `prelec`, or `ge`).

```bash
# beta = 0.95
uv run experiment.py --model eut --n-steps 5000 --max-workers 6 --beta 0.95
uv run experiment.py --model pt  --n-steps 5000 --max-workers 6 --beta 0.95
uv run experiment.py --model cpt --n-steps 5000 --max-workers 6 --beta 0.95 --cpt-weight-function kt
```

Results are saved as `.pickle` files in `data/<model>/<model>_<beta_decimals>/raw/`  
(e.g., `data/pt/pt_95/raw/` for `beta=0.95`).

### Post-Processing: Key Metrics

Computes Sen welfare, Gini coefficient, mean utility, and final agent states from the raw simulation output. Must be run before the `sen_welfare.ipynb` and `sensitivity_analysis.ipynb` notebooks.

```bash
# beta = 0.95
uv run compute_key_metrics.py --model eut --beta 0.95
uv run compute_key_metrics.py --model pt  --beta 0.95
uv run compute_key_metrics.py --model cpt --beta 0.95 --cpt-weight-function kt
```

Alternatively, use `--data-dir <path>` to point directly to any model output directory.

### Post-Processing: Frequency Analysis

Runs spectral analysis (FFT with Hann windowing and power thresholding) on agent utility trajectories to detect limit cycles. Must be run before `frequency_analysis.ipynb` and `attractors.ipynb`.

```bash
# beta = 0.95
uv run compute_trajectory_amplitudes.py --model eut --max-workers 6 --beta 0.95
uv run compute_trajectory_amplitudes.py --model pt  --max-workers 6 --beta 0.95
uv run compute_trajectory_amplitudes.py --model cpt --max-workers 6 --beta 0.95 --cpt-weight-function kt
```

Results are saved to `data/<model>/<model>_<beta_decimals>/dominant_frequencies_amplitudes.pickle`.

## 3. Specialized Experiments

These scripts run targeted parameter sweeps for specific figures.

### Bifurcation Analysis — Figure 8 (`lambda_bifurcation.py`)

Sweeps loss aversion (λ from 1 to 2.5) across PT parameter sets that exhibit high-amplitude oscillations. **Requires PT core experiment and frequency analysis (Section 2) to be completed first**, as it filters PT simulations with mean amplitude > 10.

```bash
uv run lambda_bifurcation.py --beta 0.95
```

Results are saved to `data/lambda_bifurcation/lambda_bifurcation_<beta_decimals>/raw/`.

### Alpha Versus Gamma Sweep — Figure 10 (`gamma_alpha.py`)

Runs a 24×24 grid sweep over `alpha` (wealth preference) and `gamma` (probability weighting) at fixed `lambda=2.5`, `rate=4.0`, `A=0.8`. Use `--sweep-gamma true` (default) for the CPT sweep or `--sweep-gamma false` for a PT alpha-only sweep.

```bash
# CPT sweep (alpha × gamma grid)
uv run gamma_alpha.py --beta 0.95  --sweep-gamma true

# PT sweep (alpha only)
uv run gamma_alpha.py --beta 0.95  --sweep-gamma false
```

Results are saved to `data/gamma_alpha/cpt_<beta_decimals>/raw/` or `data/gamma_alpha/pt_<beta_decimals>/raw/`.

### Probability Sweep (`probability_sweep.py`)

Runs a 15×15 grid sweep over `P_H_increase` × `P_H_decrease` for three fixed parameter regimes: `high_welfare` (α=0.30, γ=0.80), `transition` (α=0.65, γ=0.60), and `low_welfare` (α=0.70, γ=0.20).

```bash
uv run probability_sweep.py --beta 0.95
```

Results are saved to `data/probability_sweep/<regime_name>/raw/`.

## 4. Reproducing Figures (Notebook Guide)

The figures in the manuscript are generated using the Jupyter notebooks in `notebooks/` after running the core experiments outlined above. Execute them in the order below to ensure data dependencies are met.

| Notebook | Dependencies | Figures | Description |
| :--- | :--- | :--- | :--- |
| `health_cost.ipynb` | None | **2** | Plots the cost/benefit functions and model "physics". |
| `sen_welfare.ipynb` | `compute_key_metrics.py` | **3, 4, B.1., D.1-D.3, F.1, F.2** | Computes Sen Welfare metrics, Kernel Density Estimates (KDE), and marginal parameter sensitivities for all models. |
| `sensitivity_analysis.ipynb` | `compute_key_metrics.py` | **E.1** | Performs PAWN Global Sensitivity Analysis on the welfare metrics derived in the previous notebook. |
| `frequency_analysis.ipynb` | `compute_trajectory_amplitudes.py` | **5, 6, F.3** | Visualizes agent trajectories, amplitude distributions, and stability phase diagrams. |
| `attractors.ipynb` | `compute_trajectory_amplitudes.py` | **7** | Maps the spatial distribution of fixed points vs. limit cycles to visualize the "fracture" in policy space. |
| `lambda_bifurcation.ipynb` | `lambda_bifurcation.py` | **8** | Visualizes the bifurcation tipping point (transition to limit cycles) as loss aversion (λ) increases. |
| `gamma_alpha.ipynb` | `gamma_alpha.py` | **9** | Heatmaps showing the tipping point caused by probability weighting (γ) and wealth preference (α). |
| `cpt_prob_comparison.ipynb` | None | **C.1, C.2** | Comparison of probability weighting functions for different probability weighting (γ) values, as well as resulting Kernel Density Estimates. |
| `probability_sweep.ipynb` | `probability_sweep.py` | **H.1, H.2** | Results of specialized PH+ vs. PH- sweep including welfare heatmaps, as well as line plots depicting marginal welfare and average investment rate dependencies on these probabilities. |

## Data Notes

- **Raw simulation output** is stored as `.pickle` files under `data/<model>/<model>_<beta_decimals>/raw/`  
  (e.g., `data/cpt/cpt_kt_95/raw/` for CPT with Kahneman–Tversky weighting at β=0.95).
- **Processed metrics** (`sen_welfare`, `gini`, `mean_util`, `params`, `final_states`) are written by `compute_key_metrics.py` to the parent model directory.
- **Spectral results** (`dominant_frequencies_amplitudes.pickle`) are written by `compute_trajectory_amplitudes.py` to the parent model directory.
- The `data/` directory is not tracked by git. Ensure sufficient disk space if running large sweeps (1,024 LHS samples × 5,000 agents × 5,000 steps per model).
